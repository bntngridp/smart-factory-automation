# 2.1 Server Action & API untuk Produk

**Epic:** 2 — Modul Manajemen Produk (Master Data)  
**Priority:** High  
**Dependencies:** 1.1

---

## Deskripsi

Membuat fungsi asinkron (Server Actions) untuk CRUD dasar produk: mengambil daftar produk, menambah produk baru, dan memperbarui stok minimal.

## Acceptance Criteria

- [ ] `src/services/productService.ts` berisi fungsi:
  - `getProducts(): Promise<Product[]>` — mengambil semua produk, diurutkan berdasarkan `createdAt` descending.
  - `createProduct(data: CreateProductInput): Promise<Product>` — menambah produk baru.
  - `updateProduct(id: string, data: UpdateProductInput): Promise<Product>` — memperbarui data produk (khususnya `MinStock`).
- [ ] Setiap fungsi menggunakan Prisma Client dari `src/lib/db.ts`.
- [ ] `getProducts()` mendukung opsional filter `search?: string` untuk mencari berdasarkan nama produk.
- [ ] Error handling: throw error dengan pesan jelas jika produk tidak ditemukan atau validasi gagal.
- [ ] Tipe input (`CreateProductInput`, `UpdateProductInput`) didefinisikan dan diexport.

## Technical Notes

- Gunakan `"use server"` directive pada file service.
- Validasi input dasar: nama produk tidak boleh kosong, quantity tidak boleh negatif.
- Contoh struktur fungsi:

```ts
"use server"

import { prisma } from "@/lib/db"
import { revalidatePath } from "next/cache"

export async function getProducts(search?: string) {
  return prisma.product.findMany({
    where: search ? { name: { contains: search, mode: "insensitive" } } : undefined,
    orderBy: { createdAt: "desc" },
  })
}
```

## Definition of Done

- Fungsi dapat dipanggil dari komponen React Server Component.
- Data berhasil di-return tanpa error Prisma.
- Produk baru bisa ditambahkan dan stok minimal bisa diperbarui.
