# 4.1 Layout & Navigasi Utama

**Epic:** 4 — Dashboard Utama & Layout UI  
**Priority:** High  
**Dependencies:** —

---

## Deskripsi

Membuat sidebar navigation dan header yang responsif sebagai kerangka utama (root layout) aplikasi.

## Acceptance Criteria

- [ ] Root layout (`app/layout.tsx`) memiliki sidebar navigasi dan header.
- [ ] Navigasi mencakup link ke:
  - **Dashboard** → `/`
  - **Master Produk** → `/products`
  - **Log Produksi** → `/production-logs`
- [ ] Sidebar responsif: pada layar desktop tampil sebagai sidebar, pada mobile menjadi hamburger menu atau bottom navigation.
- [ ] Header menampilkan judul halaman aktif dan informasi user/session (placeholder jika belum ada auth).
- [ ] Active link di sidebar di-highlight (menggunakan `usePathname()`).
- [ ] Layout menggunakan Tailwind CSS, konsisten secara visual.
- [ ] Konten halaman berada di area utama (tidak overlap dengan sidebar).

## Technical Notes

- Struktur folder:
  ```
  app/
  ├── layout.tsx          # root layout dengan sidebar + header
  ├── page.tsx            # dashboard
  ├── products/
  │   └── page.tsx
  └── production-logs/
      └── page.tsx
  ```
- Gunakan komponen `Sidebar` dan `Header` terpisah di folder `src/components/`.
- Contoh struktur sidebar:
  ```tsx
  const navItems = [
    { href: "/", label: "Dashboard", icon: HomeIcon },
    { href: "/products", label: "Master Produk", icon: PackageIcon },
    { href: "/production-logs", label: "Log Produksi", icon: ClipboardIcon },
  ]
  ```

## Definition of Done

- Semua halaman menggunakan layout yang sama.
- Navigasi berfungsi — klik link pindah halaman tanpa error.
- Tampilan rapi dan responsif di desktop dan mobile.
