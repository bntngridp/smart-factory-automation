# 3.1 Form Input Log Produksi

**Epic:** 3 — Tracking Produksi & Pergerakan Inventaris  
**Priority:** High  
**Dependencies:** 1.1, 2.1

---

## Deskripsi

Membuat komponen form bagi operator pabrik untuk memasukkan hasil produksi harian ke tabel `ProductionLogs`.

## Acceptance Criteria

- [ ] Halaman `/production-logs` atau `/production` memiliki form input produksi.
- [ ] Form memiliki field:
  - **Produk** — dropdown yang mengambil daftar produk dari database (via `getProducts()`).
  - **Quantity** — input number, minimal 1.
  - **Nama Operator** — input text.
- [ ] Submit form memanggil server action untuk menyimpan data ke `ProductionLogs`.
- [ ] Validasi client-side: quantity wajib diisi dan > 0, operator name wajib diisi.
- [ ] Setelah submit berhasil, tampilkan pesan sukses (toast/alert) dan reset form.
- [ ] Tabel riwayat log produksi ditampilkan di bawah form (5-10 entri terakhir).

## Technical Notes

- Gunakan React `useActionState` atau `useTransition` untuk handle form submission.
- Komponen form di client component (`"use client"`).
- Server action untuk menyimpan log:

```ts
export async function createProductionLog(data: {
  productId: string
  quantity: number
  operatorName: string
}) {
  // validasi
  // insert ke database
  // revalidate path
}
```

## Definition of Done

- Form bekerja end-to-end: input → submit → data tersimpan di database.
- Produk tersedia di dropdown dari database.
- Validasi mencegah input tidak valid.
- Riwayat log produksi terbaru tampil setelah submit.
