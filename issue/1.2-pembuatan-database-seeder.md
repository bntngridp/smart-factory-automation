# 1.2 Pembuatan Database Seeder

**Epic:** 1 — Stabilisasi Infrastruktur & Koneksi Database  
**Priority:** High  
**Dependencies:** 1.1

---

## Deskripsi

Membuat script seeder otomatis untuk mengisi tabel `Products`, `ProductionLogs`, dan `InventoryMovements` dengan data dummy industri.

## Latar Belakang

Data dummy diperlukan untuk pengembangan frontend dan pengujian fitur. Data harus realistis (industri manufaktur) dan memiliki relasi yang benar antar tabel.

## Acceptance Criteria

- [ ] File `prisma/seed.ts` tersedia dan terdaftar di `package.json` pada field `"prisma": { "seed": "tsx prisma/seed.ts" }`.
- [ ] Menjalankan `npx prisma db seed` mengisi tabel **Products** dengan minimal 5 produk.
- [ ] Setiap produk memiliki relasi ke **ProductionLogs** (minimal 3 log produksi).
- [ ] Setiap log produksi memiliki relasi ke **InventoryMovements** (otomatis tipe "IN").
- [ ] Data mencakup nama produk industri realistis, contoh: "Baut Industri M8", "Sensor Suhu OMRON", "PLC Mitsubishi FX5U", "Motor Servo 400W", "Relay Industri 24V".
- [ ] Seed dapat dijalankan berulang kali (idempotent — hapus data lama sebelum insert).

## Technical Notes

- Gunakan `tsx` untuk menjalankan TypeScript seed file.
- Gunakan `prisma.$transaction` untuk insert atomic.
- Panggil `prisma.$disconnect()` di akhir.
- Pattern idempotent:
  ```ts
  await prisma.inventoryMovements.deleteMany()
  await prisma.productionLogs.deleteMany()
  await prisma.products.deleteMany()
  ```

## Definition of Done

- `npx prisma db seed` berhasil tanpa error.
- Database terisi data yang bisa diverifikasi via Prisma Studio.
- Semua relasi foreign key terpenuhi.
