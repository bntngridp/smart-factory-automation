# 1.1 Resolusi Tipe Data Prisma & Inisialisasi Klien

**Epic:** 1 — Stabilisasi Infrastruktur & Koneksi Database  
**Priority:** High  
**Dependencies:** —

---

## Deskripsi

Memperbaiki masalah case-sensitivity TypeScript pada nama model Prisma (misalnya `Products` vs `products`) dan memastikan `PrismaClient` terinisialisasi dengan benar di environment Next.js (Prisma 7).

## Latar Belakang

Proyek menggunakan Prisma 7 dengan Next.js. Beberapa kendala umum:
1. Prisma 7 memperkenalkan perubahan pada inisialisasi klien (global singleton pattern).
2. TypeScript sensitif terhadap kapitalisasi nama model — model Prisma menggunakan PascalCase (`Products`) tetapi kode panggil mungkin menggunakan camelCase (`products`).
3. `PrismaClientInitializationError` muncul ketika klien gagal terhubung ke database.

## Acceptance Criteria

- [ ] Tidak ada error merah di file `src/lib/db.ts` setelah inisialisasi.
- [ ] File `src/lib/db.ts` menggunakan global singleton pattern yang aman untuk hot-reload Next.js.
- [ ] Semua pemanggilan model Prisma di service layer menggunakan nama model yang benar sesuai skema Prisma.
- [ ] Fungsi `fetch` di service layer berhasil me-return data tanpa `PrismaClientInitializationError`.
- [ ] Koneksi database dapat diverifikasi melalui Prisma Studio atau log server.

## Technical Notes

- Gunakan pattern:
  ```ts
  const globalForPrisma = globalThis as unknown as { prisma: PrismaClient }
  export const prisma = globalForPrisma.prisma ?? new PrismaClient()
  if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma
  ```
- Pastikan `DATABASE_URL` terisi di `.env`.
- Model Prisma reference: `prisma/schema.prisma`.

## Definition of Done

- `src/lib/db.ts` bersih tanpa error.
- Semua service layer yang memanggil Prisma berfungsi normal.
- Tidak ada error Prisma saat build atau dev server.
